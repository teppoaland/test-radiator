Key Architecture Features:

  • Hybrid Trigger System: Manual execution via Azure DevOps Pipeline or automatic via GitHub commits with [test]
  • Azure Pipeline Integration: Self-hosted agent can manually trigger test execution by pushing [test] commits to GitHub
  • Multi-Platform Testing: Supports both Android (via Appium) and Windows applications (via WinAppDriver)
  • Git Storage Database: Centralized history persistence across multiple projects (appium-tests, sbsaa-tests, notepad-tests)
  • Project Isolation: Separate namespaces prevent data contamination between different test suites
  • Azure DevOps Integration: Full work item lifecycle management with automated state transitions
  • PAT Authentication: Secure token-based authentication with Azure DevOps using GitHub Secrets
  • Pytest Hook System: Automatic test result reporting to Azure DevOps via custom pytest hooks
  • Test-to-Work-Item Mapping: JSON-based mapping system linking test functions to Azure DevOps work items
  • History Management: Pre/post processing to handle Allure's malformed data injection
  • Cross-Platform Device Testing: Hardware-accelerated execution with ADB shell support and Windows app automation
  • Scalable Reporting: Automated deployment to GitHub Pages with trend analysis
  • Test Radiator Dashboard: Real-time monitoring and visualization with architecture flowchart view
  • Automated Bug Creation: Failed tests automatically create linked bug work items in Azure DevOps

Execution Flow Options:

Option 1: Azure DevOps Manual Trigger

  1. Manually run Azure DevOps pipeline from the web interface
  2. Self-hosted agent executes pipeline script
  3. Pipeline pushes empty commit with [test] keyword to target GitHub repository
  4. GitHub Actions detects [test] in commit message and triggers workflow
  5. Continue with standard test execution flow

Option 2: GitHub Direct Trigger

  1. Push code commit with [test] in commit message to GitHub
  2. GitHub Actions immediately triggers workflow
  3. Continue with standard test execution flow

Standard Test Execution Flow:

  1. Load previous test history from git storage
  2. Load test-to-work-item mappings from centralized storage
  3. Initialize Azure DevOps client with PAT authentication
  4. Initialize test radiator monitoring
  5. Prepare history files for current test run
  6. Execute pytest test suite against target platform (Android device or Windows applications)
  7. Pytest hooks automatically capture test results and send to Azure DevOps
  8. Update Azure DevOps work items with test execution status (Design â†’ Closed for passing tests)
  9. Create bugs in Azure DevOps for any failed tests with detailed error information
  10. Update radiator dashboard with real-time test status
  11. Generate Allure HTML report with historical trends
  12. Clean up malformed data that Allure injects
  13. Save updated history and radiator data back to git storage
  14. Save updated test mappings for future runs
  15. Deploy final report and radiator dashboard to GitHub Pages

Azure DevOps Integration Benefits:

  • Manual Test Triggering: On-demand test execution from Azure DevOps interface without code changes
  • Work Item Traceability: Direct links between test functions and Azure DevOps work items
  • Automated Status Updates: Test results automatically update work item states and history
  • Bug Management: Failed tests create detailed bug reports with error information and links
  • Cross-Platform Workflow: Combines Azure DevOps work management with GitHub Actions execution
  • Multi-Repository Support: Single Azure pipeline can trigger tests across multiple GitHub repositories

Radiator Dashboard Features:

  • Real-time Status: Live updates of test execution progress across all projects
  • Historical Trends: Visual representation of test success rates and performance over time
  • Multi-Project View: Consolidated dashboard showing status of appium-tests, sbsaa-tests, notepad-tests, and app-availability-check projects
  • API Integration: RESTful endpoints for external monitoring tools and integrations
  • Dark Mode UI: Modern, eye-friendly interface optimized for continuous monitoring
  • Direct Report Links: One-click access to detailed Allure reports for each project
  • Architecture Visualization: Interactive flowchart showing complete system architecture

Platform Support:

  • Android Testing: Real device automation via Appium and ADB for weather app testing
  • Windows Application Testing: Desktop automation via WinAppDriver for Notepad, Calculator, and File Explorer
  • Web Application Testing: Browser automation capabilities (extensible)
  • API Testing: REST/GraphQL endpoint testing (extensible)
